========================================================
REQUIREMENT DOCUMENT (FRD & TRD)
VILLAGE MEDICINE DELIVERY WEB APPLICATION
========================================================

1. FUNCTIONAL REQUIREMENTS (FRD)
--------------------------------------------------------
1.1 User Authentication (Random Password Flow)
- Registration: User enters Mobile Number and Name. System generates a random 10-character password and sends it via SMS.
- Login: User logs in using Mobile Number and the random password sent by SMS.
- Password Reset: User enters Mobile Number. System generates a new random password, updates the database users table, and sends it to the user via SMS.

1.2 Order Submission Form
- Area Selection: Dropdown of village areas/landmarks.
- Provider Selection: Dropdown (default: Local Shopkeeper/Gaon ka Ladka).
- Product Requirement: Text area for medicine names or symptoms.
- Photo Upload (Optional): Option to upload prescription/wrapper image.
- Delivery Timeline: Hardcoded dropdown (Today, Tomorrow, Within 3 days, Within 1 week).

1.3 Business Rules
- 7 PM Cut-off: If "Today" is selected after 7:00 PM, system automatically schedules/prompts for "Tomorrow" (since delivery starts at 7:30 PM).


2. TECHNICAL REQUIREMENTS (TRD)
--------------------------------------------------------
2.1 Frontend Architecture
- Technologies: HTML, CSS, Vanilla JavaScript.
- Hosting: GitHub Pages (100% Free).

2.2 Zero-Backend "CSV" Database Strategy
- Tool: Google Sheets + Google Apps Script.
- Flow: Frontend JS sends POST request with data -> Apps Script receives it -> Appends as a new row in a Google Sheet. 
- Benefit: The shopkeeper can view this Sheet live or download as CSV. No server costs.

2.3 Authentication & SMS Execution
- Challenge: Cannot use Firebase for custom passwords. Standard SMS needs API.
- Solution: Frontend generates the random password. Sends data to Google Apps Script. Apps Script saves it in "Users" Sheet and calls an SMS API (like Fast2SMS) to send the password.

2.4 Photo Upload Solution
- Tool: ImgBB API.
- Flow: User uploads image -> Sent to ImgBB -> Returns a URL -> This URL is saved in the Google Sheet along with the order details.
